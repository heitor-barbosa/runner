# Alterações para US-01.6: Invocar assinador.jar via HTTP

Este documento detalha as modificações necessárias para implementar a funcionalidade de invocação do `assinador.jar` via HTTP, com fallback para o modo local, conforme a US-01.6 da Sprint 3.

## 1. Arquivo: `projetos/assinador/internal/runner/runner.go`

Este arquivo foi o principal ponto de alteração, introduzindo a lógica de detecção de servidor ativo e a invocação HTTP, além de refatorar a invocação local.

### Funções `InvokeSign` e `InvokeValidate` modificadas

As funções `InvokeSign` e `InvokeValidate` foram alteradas para chamar uma nova função `invokeWithFallback`, que encapsula a lógica de tentar a invocação HTTP primeiro e, em caso de falha ou servidor inativo, recorrer à invocação local.

```go
// InvokeSign invoca o assinador.jar no modo local para criacao de assinatura.
func InvokeSign(payload map[string]interface{}) (*Response, error) {
	return invokeWithFallback("sign", payload)
}

// InvokeValidate invoca o assinador.jar no modo local para validacao de assinatura.
func InvokeValidate(payload map[string]interface{}) (*Response, error) {
	return invokeWithFallback("validate", payload)
}
```

### Função `invokeLocal` (antiga `invoke`)

A função `invoke` original foi renomeada para `invokeLocal` para clareza, e agora é chamada apenas quando o fallback é necessário.

```go
func invokeLocal(command string, payload map[string]interface{}) (*Response, error) {
	javaPath, err := findJava()
	if err != nil {
		return nil, err
	}

	jarPath, err := findJar()
	if err != nil {
		return nil, err
	}

	jsonBytes, err := json.Marshal(payload)
	if err != nil {
		return nil, fmt.Errorf("erro ao serializar parametros: %w", err)
	}

	args := []string{"-jar", jarPath, command, "--json", string(jsonBytes)}
	cmd := exec.Command(javaPath, args...) //nolint:gosec

	var stdout, stderr bytes.Buffer
	cmd.Stdout = &stdout
	cmd.Stderr = &stderr

	if err := cmd.Run(); err != nil {
		if stdout.Len() > 0 {
			return parseResponse(stdout.Bytes())
		}
		return nil, fmt.Errorf(
			"falha ao executar assinador.jar: %w\nSaida de erro: %s",
			err,
			stderr.String(),
		)
	}

	return parseResponse(stdout.Bytes())
}
```

### Nova função `invokeWithFallback`

Esta é a nova função central que decide se a invocação será via HTTP ou local.

```go
func invokeWithFallback(command string, payload map[string]interface{}) (*Response, error) {
	// Tenta invocar via HTTP se o servidor estiver ativo
	if state, err := readServerState(defaultServerPort); err == nil && isServerActive(defaultServerPort) {
		resp, httpErr := invokeHttp(command, payload, state.Port)
		if httpErr == nil {
			return resp, nil
		} else {
			// Log do erro HTTP e tenta fallback para local
			fmt.Fprintf(os.Stderr, "Erro ao invocar via HTTP, tentando modo local: %v\n", httpErr)
		}
	}

	// Fallback para invocacao local
	return invokeLocal(command, payload)
}
```

### Nova função `invokeHttp`

Responsável por realizar as requisições HTTP para o `assinador.jar` quando ele está em modo servidor.

```go
func invokeHttp(command string, payload map[string]interface{}, port int) (*Response, error) {
	jsonBytes, err := json.Marshal(payload)
	if err != nil {
		return nil, fmt.Errorf("erro ao serializar payload para HTTP: %w", err)
	}

	url := fmt.Sprintf("http://127.0.0.1:%d/%s", port, command)
	req, err := http.NewRequest(http.MethodPost, url, bytes.NewBuffer(jsonBytes))
	if err != nil {
		return nil, fmt.Errorf("erro ao criar requisicao HTTP: %w", err)
	}
	req.Header.Set("Content-Type", "application/json; charset=utf-8")

	client := &http.Client{Timeout: 5 * time.Second}
	resp, err := client.Do(req)
	if err != nil {
		return nil, fmt.Errorf("erro ao fazer requisicao HTTP: %w", err)
	}
	defer resp.Body.Close()

	body, err := os.ReadAll(resp.Body)
	if err != nil {
		return nil, fmt.Errorf("erro ao ler resposta HTTP: %w", err)
	}

	if resp.StatusCode != http.StatusOK && resp.StatusCode != http.StatusBadRequest {
		return nil, fmt.Errorf("resposta HTTP inesperada: %s - %s", resp.Status, string(body))
	}

	return parseResponse(body)
}
```
